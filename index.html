Disk Drive

Archive data on your favorite block chains

Features

Store and retrieve data on most blockchains

Use Apertus - Disk Drive (ADD) along with your favorite crypto currency wallets to immutably archive UTF8 text messages, binary files or both! ADD's open source ledger system allows you to store 1 or more files of any size on most blockchains. You can later recover the files by entering a single transaction id. Wallets must support the standard RPC calls orginally introduced by bitcoin developers. (Most currently do)

Monitor 1 or more blockchains in "realtime"

ADD converts any compatible blockchain into a media \ messaging hub. Just check the "Monitor Block Chains" box on the explore tab, sit back and watch as all data archived by other ADD users is displayed in the monitoring window.

Sign and verify archives

Signing an archive proves the archivist owns the signing address and that the data returned to the client matches the data archived by the signee. It does this by creating a SHA256 HASH of the entire archive. The HASH is passed in as a message to the wallets signing function to create a signature. The signature and signing address is then attached to the archive for verification by other clients. Signing your archives allows Apertus users to trust, follow and find archives.

Proof of existence

We added a handy tool that Creates a SHA256 hash of any file. This hash can then be archived on any compatible blockchain as UTF8 text. The transaction Id generated during the archive process can later be used to retrieve the hash Proving the file existed at the time of the archive. To use this feature click File\Insert\Proof from the main menu.

Filter unsafe extensions

By default executable files and all web scripting is either filtered or cleansed. Never open executable files from an unverified source!

Getting Started

Install the Client

Click the link at the top of this page to download the Offical Apertus - Disk Drive Client. Create a folder on your workstation and unzip the contents of the download into it. You're done!

Install a Wallet

ADD requires a full RPC enabled Wallet to function. Some clients can take days to synchronize so be patient! Here are a few of our favourites:

Full Bitcoin Wallet      Full Litecoin Wallet      Full Dogecoin Wallet     

Enable Wallet RPC Access

To enable RPC create a [CoinName].conf file using notepad and place it in the Roaming App Data file associated with your wallet.

Example: C:\Users\[Your user name]\AppData\Roaming\Bitcoin\bitcoin.conf
Example: C:\Users\[Your user name]\AppData\Roaming\Litecoin\litecoin.conf

You may need to enable the viewing of hidden folders and files to see the AppData Folder
Paste the following three lines into your [CoinName].conf file and restart the corresponding wallet. Be sure to change the credentials to something other than this example! 

server=1
rpcuser=RPC_USER_CHANGE_ME
rpcpassword=RPC_PASSWORD_CHANGE_ME

Configure ADD to use your wallets

Now that you have enabled RPC on your wallet you need to configure ADD to talk to the wallet. Open ADD by browsing to the folder you created and running the ADD.exe program.
Click on Settings \ Wallet.

Look we pre-loaded the settings for several common wallets for you!!

Select the wallet you would like to begin using.
Change the login and password to match the credentials you entered in the [CoinName].conf file.
Check the Enable Wallet box
Click the Save Button.
NOTE: Do not alter any other settings unless you really know what your doing..

Start Archiving and Exploring!

ADD Archives data by permanently moving small units of crypto currency to specialy encoded addresses that store portions of the data you are archiving inside. A unique transaction ID allows the complete archive to be reasembled later. it may take hundredes or even thousands of micro transactions to archive your data depending on the Cyrpto Currency you are using and the amount of data you are archiving. Coins used in the archive process cannot be recovered and once archived the data cannot be altered in any way. Make sure you have triple checked your data for errors before archiving.
